# 11 - Demo Script (Hackathon)

## Demo Goal
Show a consumer-friendly experience powered by an enterprise-grade policy intelligence engine.

## 4-Minute Demo Flow

### 0:00-0:30 - Problem
"Insurance drug policies are hard to read, inconsistent across payers, and change often. People and teams waste hours finding basic answers."

### 0:30-1:10 - Upload + Processing
- Upload a policy PDF.
- Show processing status moving to completed.
- Briefly show extracted structured rule card.

### 1:10-2:00 - Ask a Question
- Ask: "Does Plan A cover Drug X?"
- Show answer with plain wording.
- Open citation card showing source snippet and page.

### 2:00-2:40 - Compare Plans
- Select Drug X.
- Show side-by-side compare table for two plans.
- Highlight one key difference (for example step therapy required by one plan only).

### 2:40-3:20 - Show Change Tracking
- Open policy version diff.
- Show added/removed/modified criteria with citations.

### 3:20-3:50 - Voice Call Agent
- Place/receive call.
- Ask one practical question.
- Show response and post-call summary in UI.

### 3:50-4:00 - Close
"We combine transparency, trust (citations), and automation (auto-update agent) so users can act on current policy data."

## Backup Plan (If Live Integration Fails)
- Use prerecorded call clip.
- Use cached query response with visible citations.
- Use prepared before/after diff dataset.

## Judge-Friendly Language
- Prior authorization: "insurance pre-approval"
- Coverage criteria: "rules you must meet"
- Diff: "what changed between old and new policy"

## Success Signals to Emphasize
- Time saved vs manual review
- Explainable answers (source-backed)
- Works for both patient mode and professional mode
